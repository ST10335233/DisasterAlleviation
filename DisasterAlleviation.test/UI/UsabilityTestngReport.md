Usability Testing Report – Disaster Alleviation Application
1. Purpose

Evaluate the usability of the Disaster Alleviation application.

Assess how easily users can navigate the interface, perform tasks, and understand content.

Identify areas for improvement to enhance user satisfaction.

2. Scope

Tested core functionalities for volunteers, admins, and donors.

Focused on navigation, form completion, readability, and feedback messages.

Gathered qualitative feedback from fictitious users.

3. Test Participants

3–5 fictitious users representing different roles:

Volunteers

Admins

Donors

Participants performed common tasks to simulate real-world use.

4. Test Scenarios

Volunteer Registration

Navigate to registration page and complete the form.

Submit valid and invalid input to test validation messages.

Incident Reporting

Navigate to report incident page.

Fill in incident details and submit.

Test error handling for missing or incorrect fields.

Task Assignment (Admin)

Assign tasks to volunteers.

Check if assigned tasks appear correctly.

Donations

Submit donations with valid and invalid inputs.

Check status updates for each donation.

Navigation

Move between pages using menus and links.

Test Back and Forward browser buttons.

5. Observations and Feedback

Users found navigation intuitive and menus easy to use.

Forms were easy to complete; input fields were clear.

Validation messages were visible but some users suggested more descriptive errors.

Page layout and content readability were good, but some minor alignment issues were noted.

Mobile responsiveness worked reasonably well; some users suggested slight improvements for smaller screens.

6. Issues Identified

Error messages could be more specific (e.g., “Enter a valid email” instead of generic message).

Minor visual alignment issues on donation and task assignment pages.

Some buttons and links could use consistent styling for better clarity.

7. Recommendations

Improve error messages for better clarity.

Adjust layout alignment in forms for a cleaner look.

Standardize button styles across all pages.

Consider optimizing mobile responsiveness for smaller screens.

8. Conclusion

Overall, the Disaster Alleviation application is user-friendly and functional.

Users were able to complete tasks without confusion.

Feedback provides actionable insights for future improvements in usability, layout, and error handling.
